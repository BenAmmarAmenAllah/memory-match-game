## Architecture Principles

### 1. **Separation of Concerns**
Each component has a single, well-defined responsibility:
- **UI Components**: Handle rendering and user interactions
- **Hooks**: Manage state logic and side effects
- **Utils**: Pure functions for data manipulation
- **Contexts**: Global state distribution

## Design Decisions

### Why React Context over Redux?
- The application state is relatively simple
- No need for middleware or time-travel debugging
- Reduces boilerplate and dependencies
- Context + useReducer provides similar patterns

### Why Separate Audio Context?
- Audio state is independent of game state
- Allows music to persist across game resets
- Enables independent testing of audio features
- Cleaner separation of concerns

### Why Custom Hooks?
- Encapsulates complex logic
- Makes components cleaner and more focused
- Enables reuse across components
- Easier testing of business logic

## Performance Considerations

1. **Memoization**: Use `React.memo()` for Card components to prevent unnecessary re-renders
2. **Lazy Loading**: Overlay screens (Win/Lose) can be lazily imported
3. **Audio Preloading**: Preload sound effects during initial load
4. **CSS Transforms**: Use GPU-accelerated transforms for flip animations
